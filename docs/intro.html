Governança de IA — conceitos essenciais

Uma introdução ao que significa governar agentes de IA, por que isso importa, e como as ferramentas deste ecossistema se encaixam.

Fundamentos

O que é governança de agentes de IA?

Agentes de IA como Claude, Copilot e Cursor escrevem código, tomam decisões e executam ações no seu repositório. São ferramentas poderosas — e sem regras claras, cada agente interpreta o trabalho à sua maneira, sem registro, sem limites de escopo e sem previsibilidade.

Governança de IA é a camada de contratos, limites e registros que garante que agentes (e humanos) sigam os mesmos processos. Em vez de confiar que o agente "entenda" as regras pelo contexto, você as torna explícitas, executáveis e auditáveis.

O resultado: trabalho com IA tão previsível quanto um pipeline de CI — onde qualquer pessoa (ou agente) que entrar no projeto sabe exatamente o que pode, o que não pode, e como documentar o que fez.

Público

Para quem é isso?

Governança não é só para grandes times. Ela resolve problemas reais em qualquer escala.

🧑‍💻

Dev solo

Você usa vários agentes de IA em projetos diferentes e quer que todos sigam as mesmas convenções — sem repetir instruções toda hora.

👥

Time de desenvolvimento

O time adotou IA, mas cada membro configurou seu agente de forma diferente. Governança cria um contrato compartilhado que todos os agentes seguem.

🏢

Engenharia corporativa

Você precisa de rastreabilidade, limites de escopo e trilha de auditoria para atender políticas de segurança e conformidade com uso de IA.

AI Governance — core concepts

An introduction to what governing AI agents means, why it matters, and how the tools in this ecosystem fit together.

Fundamentals

What is AI agent governance?

AI agents like Claude, Copilot, and Cursor write code, make decisions, and execute actions inside your repository. They're powerful tools — and without clear rules, each agent interprets its work differently, with no log, no scope limits, and no predictability.

AI governance is the layer of contracts, limits, and records that ensures agents (and humans) follow the same processes. Instead of hoping the agent will "pick up" the rules from context, you make them explicit, enforceable, and auditable.

The result: AI-assisted work as predictable as a CI pipeline — where anyone (or any agent) joining the project knows exactly what they can do, what they can't, and how to document what they did.

Audience

Who is this for?

Governance isn't only for large teams. It solves real problems at any scale.

🧑‍💻

Solo developer

You use several AI agents across different projects and want them all to follow the same conventions — without repeating instructions every time.

👥

Development team

Your team adopted AI, but each member configured their agent differently. Governance creates a shared contract that every agent follows.

🏢

Enterprise engineering

You need traceability, scope limits, and an audit trail to satisfy security policies and compliance requirements around AI use.

Reference

Concepts glossary

AI agent An AI tool that acts autonomously inside a repository — writing code, creating files, or running commands. E.g.: Claude Code, GitHub Copilot, Cursor.
contract An explicit set of rules that defines what an agent can and cannot do in a project. Typically a Markdown file (AGENTS.md) read by the agent at the start of each session.
policy pack A portable collection of contract and configuration files (the AI-Agents project) that can be installed into any repository to provide immediate governance.
session A scoped unit of work — typically mapped to an issue. It has a start, an end, and a handoff file that records what was done.
handoff A document generated at the end of a session that summarizes decisions, changes, and work state. Allows another agent (or human) to resume without losing context.
issue workflow A structured folder and file convention for tracking tasks. Each issue has its own directory with a brief, progress log, and handoff — auditable and platform-independent.
doctor The governancekit doctor command that validates whether a repository's governance scaffold is correct — checks required files, permissions, and configuration.
scope limit An explicit restriction in the contract that prevents the agent from touching files, folders, or systems outside the current session's scope.

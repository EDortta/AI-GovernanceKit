AI GovernanceKit é um toolkit de orquestração local que transforma regras de governança de repositório em fluxos de trabalho executáveis — utilizável por CLIs, agentes de IDE, servidores MCP e jobs de CI.

Começar → Ver no GitHub

O problema

Regras de governança existem.
Elas simplesmente não são aplicadas.

Times escrevem arquivos AGENTS.md, docs de fluxo de trabalho e contratos de código — depois assistem agentes ignorá-los porque não existe runtime para validá-los e aplicá-los.

📋

Regras são copiadas, não aplicadas

O Markdown de governança é duplicado entre repositórios mas nenhuma ferramenta verifica se os agentes realmente o seguem. A deriva é invisível até que algo quebre.

🤖

Agentes interpretam regras de formas diferentes

Claude, Codex, Cursor e assistentes de IDE executam as mesmas instruções de maneiras diferentes. Não existe camada agnóstica de ferramenta para normalizar o comportamento.

🔍

Nenhuma trilha de auditoria para trabalho com IA

Quando um agente modifica código, fecha uma issue ou faz push de um branch, raramente existe registro do contexto que ele tinha, quais gates verificou ou qual evidência coletou.

Como funciona

Uma camada de runtime ao redor dos seus agentes existentes

GovernanceKit não substitui seu agente de código. Ele o envolve com um fluxo de trabalho reproduzível: restaurar contexto, validar gates, executar verificações, coletar evidências, encerrar sessões.

Camada 1

Policy Pack

Contratos legíveis por humanos — AGENTS.md, guias de papel, limites e visão geral do projeto — definem o "o quê e por quê" para cada agente e programador.

Camada 2

CLI

Comandos para doctor, resume, start-work, validação e session-close que qualquer agente ou humano pode invocar.

Camada 3

Runtime Core

Lógica de orquestração que carrega políticas, avalia gates e registra evidências de auditoria de forma reproduzível e agnóstica a ferramentas.

Camada 4

Integrações

Servidor MCP opcional, extensão de IDE, helpers para GitHub e Jira, e hooks de CI que conectam o runtime ao seu toolchain existente.

Quick start

Em funcionamento em 60 segundos

Instale o pacote e execute doctor em qualquer repositório que siga o scaffold de governança do AI-Agents.

  • Valida AGENTS.md, README e arquivos de handoff
  • Verifica flags de prontidão em software-overview e limits
  • Confirma estrutura de issue ativa e próximo passo do RESUME
  • Detecta caminhos de arquivos secretos rastreados antes que vazem
terminal 
# Instale pelo GitHub
pip install git+https://github.com/EDortta/AI-GovernanceKit.git

# Ou clone e instale em modo editável
git clone https://github.com/EDortta/AI-GovernanceKit.git
pip install -e ./AI-GovernanceKit

# Valide seu scaffold de governança
governancekit doctor

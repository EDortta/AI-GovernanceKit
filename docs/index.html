The problem

Governance rules exist.
They just aren't enforced.

Teams write AGENTS.md files, workflow docs, and coding contracts — then watch agents ignore them because there is no runtime to validate and enforce them.

📋

Rules are copied, not enforced

Governance Markdown is duplicated across repositories but no tool checks whether agents actually follow it. Drift is invisible until something breaks.

🤖

Agents interpret rules differently

Claude, Codex, Cursor, and IDE assistants each execute the same instructions in different ways. There is no tool-agnostic layer to normalize behavior.

🔍

No audit trail for AI work

When an agent modifies code, closes an issue, or pushes a branch, there is rarely a record of which context it had, which gates it checked, or what evidence it collected.

How it works

A runtime layer around your existing agents

GovernanceKit does not replace your coding agent. It wraps it with a reproducible workflow: restore context, validate gates, run checks, collect evidence, close sessions.

Layer 1

Policy Pack

Human-readable contracts — AGENTS.md, role guides, limits, and project overview — define the "what and why" for every agent and programmer.

Layer 2

CLI

Commands for doctor, resume, start-work, validation, and session-close that any agent or human can invoke.

Layer 3

Runtime Core

Orchestration logic that loads policy, evaluates gates, and records audit evidence in a reproducible, tool-agnostic way.

Layer 4

Integrations

Optional MCP server, IDE extension, GitHub and Jira helpers, and CI hooks that connect the runtime to your existing toolchain.

Quick start

Up and running in 60 seconds

Install the package and run doctor against any repository that follows the AI-Agents governance scaffold.

  • Validates AGENTS.md, README, and handoff files
  • Checks readiness flags in software-overview and limits
  • Confirms active issue structure and RESUME next step
  • Detects tracked secret-file paths before they leak
terminal 
# Install
pip install ai-governancekit

# Or run directly from the repo
python3 -m pip install -e .

# Validate your governance scaffold
governancekit doctor

✓ AGENTS.md exists
✓ README.md exists
✓ handoff.md exists
✓ project_context_ready: yes
✓ limits_ready: yes
✓ Active epic found
✓ RESUME next step present
✓ No tracked secret files

All checks passed.
